# ROADMAP

## Product Direction

Caledit is designed as a production-grade, self-hosted calendar design platform.
The development strategy is incremental and outcome-driven:

- Deliver usable vertical slices in each phase
- Prioritize reliability and print-quality output
- Keep architecture maintainable for long-term evolution

## Phases

### Phase 0 — Infrastructure
- Monorepo setup
- Frontend + backend foundations
- Docker and database baseline

### Phase 1 — Authentication and User Management
- Secure login/logout/refresh
- Protected routes
- Admin user management

### Phase 2 — Projects
- Project CRUD
- 12-month project structure
- Dashboard and project overview

### Phase 3 — Asset Library
- Upload, folders, thumbnails
- Search and organization

### Phase 4 — Calendar Grid Editor
- Editable day grid
- Typography and color controls
- Save and autosave

### Phase 5 — Holidays and Events
- Regional holiday support
- Recurring and custom events

### Phase 6 — Visual Canvas Editor
- Full-page Fabric.js editing
- Layers, text, stickers, backgrounds
- Undo/redo and zoom

### Phase 7 — Advanced Day Cells
- Per-cell images, stickers, and labels

### Phase 8 — Templates
- Reusable style templates
- Apply-to-all and month-level overrides

### Phase 9 — Export
- PDF/PNG export pipeline
- High-quality rendering and job tracking

### Phase 10 — UX and Production Readiness
- Onboarding
- Friendly error handling
- Rate limiting and performance improvements
- Test coverage and operational hardening

## Current focus

- Stabilization and operational excellence
- Documentation and test expansion
- Production deployment ergonomics
